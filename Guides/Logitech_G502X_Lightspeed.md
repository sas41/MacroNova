# Logitech G502 X Lightspeed

This guide covers setup for G502 X Lightspeed with the new device-first MacroNova flow.

## 1) Ensure buttons emit events

Before using MacroNova capture, confirm Linux receives button events:

```sh
sudo evtest
```

Pick your G502 event device and press side/sniper buttons. You should see `EV_KEY` events (`BTN_SIDE`, etc.).

If events do not appear, fix mouse firmware/profile first (Piper/Solaar/device memory profile).

---

## 2) Add the device in MacroNova GUI

1. Open `macronova-gui`.
2. Go to `Bindings` tab.
3. Click `+ Add device`.
4. Select the Logitech receiver/device entry from the discovered list.

This creates a `[[devices]]` config entry with its `mouse_path` (and optional `kbd_path`).

---

## 3) Add binding and capture

1. Under the added device section, click `+ Add binding`.
2. Click `Capture` on that binding row.
3. Press the target G502 button.

Capture is scoped to that device, so other mice/keyboards will be ignored while capturing that row.

---

## 4) Assign scripts

Choose `on_press` (and optional `on_release`) script from the dropdown.

Example binding value (auto-generated by capture):

```toml
button = "usb-Logitech_USB_Receiver::usb-Logitech_USB_Receiver-event-mouse/key0x0113"
```

---

## 5) Optional intercept mode

Enable `Virtual Mode` in Settings and then enable `Intercept` on selected bindings if you want MacroNova to consume those button events instead of letting apps/OS see them.

---

## Piper note

Some Logitech profiles can remap or disable physical buttons. If capture is inconsistent, ensure button actions in Piper map to normal button click outputs (not disabled/special firmware-only actions).
